One POST /v1/audio/speech returns the MP3 and the word-by-word speech marks in the same response. Open DevTools, press the button, watch the call land in the Network tab.

    The browser posts to /v1/audio/speech on the local server. The server adds the Authorization header from SPEECHIFY_API_KEY and forwards the request to api.speechify.ai. Same path, same body, same response — the key just stays off the client. That is also how you would wire this in a real app.
